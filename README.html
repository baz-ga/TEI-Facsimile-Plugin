Samples for the oXygen SDK

The oXygen SDK allows you to extend or create new applications based on the oXygen XML editing technology.

The contents of this project

Main sample sub-projects:
oxygen-sample-applet This project is a sample application of using an oXygen XML Author Component for building an applet.
oxygen-sample-eclipse-plugin You can use this as a starting project for creating an Eclipse IDE plug-in that extends the oXygen XML Editor plug-in for Eclipse.
oxygen-sample-framework You can use this as a starting project for creating an Java extensions needed by an Author customization for a specific type of XML documents.
oxygen-sample-plugins You can find here a set of plugins for the oXygen XML Editor standalone distribution.
oxygen-sample-webapp Starting with version 16.0, oXygen can be deployed sever-side, as a web application. The editing logic is kept on the server, while the document UI is delivered to the browser using HTML5 and JQuery.

Resources sub-projects:

bundle-frameworks Packs the frameworks you need to deploy to the applet or to the webapp.
bundle-options Packs the default oXygen options you need to deploy to the applet or to the webapp.
bundle-plugins Packs the plugins you need to deploy to the applet or to the webapp. Uses all the plugins from the oxygen-sample-plugins project.
bundle-samplefiles Packs the samples you need to deploy to the applet or to the webapp.

Keep what you need

This sample project is modular. You can keep any of the main sample sub-projects and its dependencies and delete the rest. Make sure you update also the pom.xml file.

For instance if you want to develop a web application you could keep the oxygen-sample-webapp and delete from the filesystem the other oxygen-sample-* projects. Since this project depends on the other bundle-* projects make sure you keep them.

How to build the project

All the subprojects, except for the oxygen-sample-applet can be built directly. The applet project needs extra configuration, in order to be able to sign and deploy the applet to a web server.

So, running the command
mvn install

will build all artifacts, but will fail building the applet.
